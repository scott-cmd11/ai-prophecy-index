Priority Findings

AreaFinding and Acceptance Criteria
Product clarity The hero explains the ledger, but the scoring method needs a compact rubric for confirmed, unfolding, too early, incorrect, confidence, and last-reviewed.
Mobile usability At 390px there is no overflow, but tap targets are too small. Core controls should reach a practical minimum height while retaining mono editorial styling.
Sticky stats The bar avoids zero states, but mobile wording should name hit rate clearly and expose open/resolved status without implying filtered counts.
Filters Pressed state is mostly colour. Add non-colour cues, an all-topics reset, and a clearer disabled state for the final active thinker.
Events and evidence Events and predictions are grouped by year, but the relationship is only implied. Event cards should explicitly invite comparison with same-year predictions.
Source trust Prediction evidence is AI-assisted, but the inline assessment cards are not labelled as such. Add a small local disclosure without weakening the existing legal notice.
Profiles Back-to-tracker links should consistently return to the tracker. "Photo" links often point to profile/source pages and should be relabelled.
Metadata Root metadata is basic. Add canonical/Twitter metadata plus robots and sitemap routes without inventing factual claims.

Implementation Slices

Slice 1 - Trust and scoring clarity

  • Add a compact scoring rubric near the tracker method block.
  • Add AI-assisted assessment labelling in expanded prediction evidence.
  • Add event comparison/source-trust cues without changing event facts or prediction scores.

Slice 2 - Mobile interaction polish

  • Increase filter, profile, source, and disclosure controls to practical touch sizes.
  • Improve sticky stats wording and status grouping on mobile.
  • Add non-colour active cues and all-topics/reset affordances.

Slice 3 - Public route and metadata polish

  • Align profile navigation and source-link wording.
  • Add sitemap and robots route files using the current route set.
  • Update metadata and README only to match current public behaviour.
